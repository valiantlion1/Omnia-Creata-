🎨 Claude Code'a Frontend UI 
OmniaCreata Studio frontend iyileştirmesi yapacaksın. Proje: apps/studio/web/src

Öncelik sırasıyla şu görevleri yap:

── 1. pages/Create.tsx ──
- State'te zaten var olan negative_prompt, seed, steps, cfg_scale alanlarını UI'a bağla. Prompt textarea'nın altına sağa yaslanmış küçük bir "Advanced" butonu ekle (SlidersHorizontal ikonu). Tıklanınca animate-slide-up ile açılan compact drawer: negative prompt textarea, seed input (yanında 🎲 rastgele butonu), steps slider (10-50, default 28), cfg_scale slider (1-15, default 6.5). Drawer kapalıyken negative_prompt doluysa veya seed değişmişse butonda küçük dot indicator göster.
- Her model için dropdown'da küçük info tooltip ekle (title attribute yeterli): FLUX.1 Schnell="En hızlı, 4 adımda üretir", SDXL Base="Dengeli kalite ve hız", RealVis XL="Fotorealistik çıktılar için", Juggernaut XL="Yüksek detay ve derin renk"
- creditSummary.remaining < 10 ise aspect ratio butonlarının üstünde amber uyarı banner'ı göster

── 2. pages/Chat.tsx ──
- composeModes = ['Think', 'Vision', 'Edit'] zaten kodda var. Input alanının üstüne pill group mode switcher ekle. Aktif mod gradient (--primary→--accent), inaktif bg-white/[0.04]. İkonlar: Think=Brain, Vision=Eye, Edit=Wand2
- Hiç mesaj yokken ekran ortasında empty state: başlık "Ne yaratmak istiyorsun?", 3 quick-start chip: "Prompt yaz" / "Görsel yükle" / "Stil öneri" — tıklayınca ilgili mode'a geçip textarea'ya focus olsun

── 3. components/StudioShell.tsx ──
- Sidebar en altına (logout üstüne) kredi widget: [Zap ikon] "X kredi" + "/subscription" linki. remaining < 10 ise text-amber-400
- Aktif nav item'a renk vurgusunun yanı sıra sol kenarda border-l-2 accent çizgisi ekle

── 4. pages/MediaLibrary.tsx ──
- "No images yet" empty state'ini zenginleştir: büyük muted ikon, başlık, açıklama, "/create" linkli gradient buton
- processing status'lu görsel kartlarına shimmer placeholder (animate-shimmer class zaten tailwind.config.js'de tanımlı)

── 5. pages/Elements.tsx ──
- "Create style" kartına tıklayınca "Yakında — Creator planında geliyor" modal'ı aç (fixed overlay + dark card)
- Her style kartına hover overlay "Stil uygula →" + kısa description ekle

── 6. pages/Settings.tsx ──
- Kredi göstergesinin altına ince progress bar ekle (Free=60 max, Pro=1200 max). %20 altı kırmızı, %20-50 sarı, üstü gradient

⚙️ Codex'e Backend 
OmniaCreata Studio backend hardening ve kapasite artırımı. Proje: apps/studio/backend

Aşağıdaki görevleri temkinli, aşamalı ve güvenli şekilde yap. Her değişiklik için mevcut test suite'i çalıştır.

── 1. Provider Resilience ──
- Provider çağrılarında circuit breaker pattern uygula: ardışık 5 başarısız istek sonrası provider 60 saniye devre dışı bırakılsın, sonra half-open state ile test isteği atılsın
- Her provider için retry logic: max 3 deneme, exponential backoff (1s, 2s, 4s), timeout 30s
- fal ve runware provider'larının not_configured durumunu graceful handle et: bu provider'lar queue'ya hiç girmesin, sadece huggingface/pollinations/demo aktifken iş kabul etsin

── 2. Generation Queue Hardening ──
- Job'ların timed_out durumuna düşme eşiğini kontrol et, 120s altındaysa 180s'ye çıkar
- Kuyrukta bekleyen job sayısı 50'yi geçerse yeni gelen isteklere 429 dön, response body'de "queue_full: true" ve tahmini bekleme süresi ekle
- Orphaned job cleanup: 24 saatten eski ve terminal olmayan job'ları her gece 03:00'da cron ile timed_out'a çek ve logla

── 3. Security Pass ──
- Tüm generation endpoint'lerinde rate limiting kontrol et: authenticated user başına dakikada max 10 istek, IP başına dakikada max 20 istek
- Input validation: prompt max 2000 karakter, negative_prompt max 500 karakter, steps 1-50 arasında, cfg_scale 1-20 arasında — bu sınırları aşan isteklere 422 dön
- SQL injection ve prompt injection için mevcut sanitization'ı kontrol et, eksikse ekle

── 4. Observability ──
- Her generation job için şu metrikleri logla: provider adı, model adı, başlangıç zamanı, bitiş zamanı, süre (ms), status, hata kodu (varsa)
- /health endpoint'ine provider bazlı detay ekle: her provider için son 5 dakikadaki success_rate ve avg_latency_ms göster
- Failed job'larda error mesajını hem DB'ye kaydet hem de structured log'a yaz

── 5. Postgres Production Path ──
- DB bağlantısında connection pool min=2 max=10 olarak ayarla
- Long-running query'ler için statement_timeout=30000 ekle
- Index kontrolü: generations tablosunda (identity_id, status, created_at) composite index var mı? Yoksa migration yaz

Kurallar:
- Her değişiklik backward compatible olsun, mevcut API contract bozulmasın
- Yeni endpoint ekleniyorsa OpenAPI spec'e de ekle (varsa)
- Secrets ve credentials hiçbir log'a yazılmasın
- Her kritik değişiklik için unit test yaz veya mevcutu güncelle
