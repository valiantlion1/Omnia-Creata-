# OmniaCreata Studio UI AI Brief

Last updated: 2026-04-13

## Purpose

This brief is for AI design tools such as Figma Make and Google Stitch.

Use it to generate OmniaCreata Studio UI concepts without drifting into generic AI dashboard design.

## Product Summary

OmniaCreata Studio is a premium AI visual product for everyday users, creators, entrepreneurs, marketers, students, founders, and professionals who need visual output fast.

It should feel:

- premium
- modern
- useful
- beautiful
- intuitive
- powerful without being intimidating

## Core Product Split

The product has two clearly separate surfaces:

### Chat

Chat is a multimodal conversational workspace.

Users should be able to:

- talk to the AI
- upload images
- upload files
- ask for prompt help
- ask for edits
- ask for inpainting
- ask for variations
- ask for analysis and transformation workflows

Chat is a major differentiator and should be treated as a flagship surface.

### Create

Create is the direct deterministic generation workspace.

Users should be able to:

- write prompts directly
- choose models
- choose styles
- pick aspect ratios
- configure generation controls
- run focused image generations quickly

Chat and Create must remain clearly distinct because they solve different user intents.

## What The UI Should Achieve

- explain the product quickly
- help normal people understand what they can do on the first screen
- make upload, transformation, and generation actions feel effortless
- preserve strong layout and interaction quality
- feel like a real launch-ready product, not a concept render

## Visual Direction

Preferred palette direction:

- cream / soft white
- ice blue / mist blue
- muted graphite / slate / charcoal support tones
- premium contrast without looking corporate

Visual goals:

- clean but not sterile
- premium but not cold
- detailed but not crowded
- strong spacing and placement
- high-end interaction quality

## Must Feel Like

- a calm premium product
- an app-first experience
- a real tool for serious use
- modern without trend-chasing
- polished and trustworthy

## Must Not Feel Like

- a generic SaaS dashboard
- a purple startup template
- a crypto interface
- a cyber control room
- an enterprise admin panel
- a chart museum
- a marketing landing page inside an app
- a glassmorphism-heavy concept

## Hard Rules

- no giant hero section inside the app
- no filler KPI cards
- no random charts unless they serve a real product reason
- no decorative right rail without real value
- no heavy gradient styling as the core visual language
- no oversized rounded corners everywhere
- no fake luxury glow effects
- no over-branded luxury copy
- no visual gimmicks replacing hierarchy
- no dashboard card mosaic as the default layout

## Layout Rules

- clear application shell
- obvious navigation
- strong main workspace
- secondary panels only if justified
- progressive disclosure: show the primary action, current context, and next useful action first
- mobile should feel intentionally recomposed, not blindly stacked from desktop

## Interaction Rules

- uploading should feel easy and central
- generation controls should feel clear and direct
- the difference between Chat and Create should be obvious immediately
- empty states should feel polished and useful
- loading states should feel calm and premium
- actions should feel fast and low-friction

## Copy Rules

Copy should be:

- calm
- clear
- competent
- trustworthy

Copy should not be:

- mystical
- cinematic for no reason
- startup-slogan heavy
- fake futuristic
- over-branded

## Accessibility

- strong contrast
- visible focus states
- clear click targets
- reduced-motion safe behavior
- meaning must not depend on color alone

## Recommended Deliverable

Generate:

- one desktop app view
- one mobile adaptation
- a flagship Chat surface
- a clearly separate Create entry or mode
- one active or selected state
- one empty or loading state

## Brand Asset

Use the attached official Omnia Creata logo asset as the brand reference.
Do not redesign the logo or invent a new mark.
