# Working Rules

## Session default
When working on Organizer, default to this product identity:
- mobile-only
- Android-first
- phone file manager
- local-first
- premium but grounded

## Always optimize for
- fast real-device usefulness
- safe file actions
- clear navigation
- believable MVP scope
- releaseable increments

## Do not drift into
- cloud-first planning
- AI-first planning
- desktop mental models
- sci-fi visual direction
- enterprise storage product language
- document-suite scope creep

## Default implementation priority
1. Browse
2. file actions
3. Search
4. Storage
5. Recycle Bin
6. releaseability

## When to pause and re-decide
Only pause for explicit product decisions if one of these changes:
- name shown in store
- canonical Phase 1 navigation
- package name
- MVP scope boundary
- release channel strategy

## What can be done without re-asking
- rewrite old capture/task code toward file-manager direction
- create or improve docs inside `apps/organizer`
- add versioning and release documentation
- remove safe generated build output inside Organizer

## What should not be touched casually
- other products in the monorepo
- Play signing secrets
- package name after store rollout starts
- Phase 2 or Phase 3 features while MVP is unstable
