# OmniaOrganizer Docs Hub

This folder is the developer-facing documentation entrypoint for OmniaOrganizer.

Its job is to:
- point humans and AI agents to the canonical product truth
- separate active planning from historical planning packs
- keep the Android-first product story readable without hiding useful older material

## Current product role

- Portfolio role: incubation / secondary review
- Product posture: mobile-only, Android-first, local-first
- Active implementation path: `mobile/android`
