# DESIGN_PRINCIPLES.md - Grounded Mobile File Manager Design

## Purpose
Protect OmniaOrganizer from drifting into over-designed concept art or bland
legacy utility UI.

## Core design stance
The app should feel:
- mobile-native
- premium but grounded
- calm
- capable
- trustworthy
- useful before it is flashy

## It must not feel like
- a sci-fi control center
- a cyber dashboard
- a cloud drive clone
- a PDF workstation
- an enterprise admin panel
- a legacy Android toolbox

## Visual principles
- surface hierarchy over heavy borders
- restrained accent use
- strong readability
- deliberate whitespace
- clear list and card hierarchy
- motion only where it improves understanding

## Interaction principles
- thumb-friendly layouts
- obvious primary actions
- long press for selection in list contexts
- safe destructive patterns
- consistent bottom navigation
- bottom sheets for secondary actions, not for hiding core navigation

## Browse principles
- file and folder scanning must be fast to read
- metadata should support the action, not dominate the screen
- list/grid modes should feel native, not gimmicky

## Storage and cleanup principles
- insight before action
- explanation before deletion
- confidence before automation

## Brand principles
- OmniaOrganizer is the everyday product name
- Omnia Creata appears as parent brand in splash, about, and selected brand
  moments
- branding must support utility, not overpower it

## Copy principles
- plain and confident language
- no fantasy wording
- no "digital sanctuary", "curation", "galactic control", or other concept
  copy that weakens the grounded utility promise
