# PRODUCT_METRICS.md - Product Success Signals

## Purpose
Define what "better" means for OmniaOrganizer beyond shipping more screens.

## North-star direction
OmniaOrganizer should increase the user's sense of control over phone storage.

## Phase 1 success signals
- users can browse and act on files without confusion
- search is used successfully on real content
- delete-to-trash is understood and trusted
- core flows are stable enough for repeated personal use

## Phase 2 success signals
- users discover useful cleanup opportunities
- storage insights change user behavior
- smart views are revisited because they are genuinely useful

## Trust metrics
- recycle-bin restore rate after delete actions
- delete-confirmation abandonment rate
- failed move/copy rate
- permission grant completion rate

## Utility metrics
- browse sessions that end in a successful file action
- search sessions with a result open or show-in-folder action
- storage visits that lead to cleanup review or category drill-in

## Guardrail metrics
- crash-free core flow rate
- large-library responsiveness
- battery impact from indexing work
- support complaints related to missing files or unsafe delete

## What not to optimize for early
- vanity AI usage
- cloud attachment metrics
- document-suite engagement

## Interpretation rule
If a metric improves by adding clutter, hidden risk, or phase drift, it does
not count as real product progress.
