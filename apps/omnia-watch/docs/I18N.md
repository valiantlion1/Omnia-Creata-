# Internationalization

## Current locales

- `en`: default product language
- `tr`: Turkish support from day one

## How it works

- locale dictionaries live in `packages/i18n`
- middleware redirects non-localized paths to a preferred locale
- locale preference can be stored in the `omnia-watch-locale` cookie
- web routes use `/:locale/...` structure

## Principles

- English-first content organization
- Turkish shipped alongside the first foundation
- future locales should be additive, not retrofit work
- shared labels and navigation copy are centralized

## Current implementation status

- public site localized
- dashboard headings and navigation localized
- language switching wired in the UI
- schema supports locale preferences

## Next steps

- localize remaining page-level helper strings that still live inside page components
- add localized validation and error copy
- sync account locale preferences between SaaS and companion onboarding
