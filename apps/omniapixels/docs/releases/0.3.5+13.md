# OmniaPixels 0.3.5+13

Date: 2026-04-27

## Status

Honesty and Android-readiness release-test build.

## Changed

- Upscale modes are now honest local modes: `Fast 2x` and `Enhanced 2x`.
- Removed the visible `AI Experimental` path until a real bundled model is proven on devices.
- Long operations now show a blocking overlay with the current operation name, local render label, elapsed seconds, and an indeterminate progress bar instead of a contextless spinner.
- Settings was cleaned up to show useful Device, Library, Build, and Diagnostics sections.
- Removed fake/planned settings rows such as `Neural model`.
- Gallery-selected upscale now reads the best available original source instead of relying on the editor preview bytes.
- Upscale render now downscales oversized inputs before 2x so the final output stays capped without a wasteful huge intermediate image.
- Saved the GPT image editor concept reference at `design/concepts/editor-premium-gpt-image-2026-04-27.png`.

## Device strategy

- Android system picker remains the preferred fast access path across device vendors.
- Custom album grid stays as an in-app fallback and diagnostics surface, not the only access path.
- Real proof must expand beyond one phone: emulator profiles, low/mid/high memory devices, Android 12-16, limited photo permission, and large libraries.

## Still needs device proof

- Install on Poco X6 Pro and at least one lower-memory Android profile.
- Verify: picker open, gallery fallback, crop ratio preview, Fast 2x, Enhanced 2x, save/share, Settings diagnostics, and long-operation overlay behavior.
