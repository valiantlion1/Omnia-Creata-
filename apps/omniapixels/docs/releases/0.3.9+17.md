# OmniaPixels 0.3.9+17

Date: 2026-04-28

## Theme

Crop control hardening.

## What changed

- Added crop position state to the editor model: horizontal and vertical crop focus now travel through preview, undo/redo, presets, reset, export, and upscale render paths.
- Fixed fixed-ratio crop export so `1:1`, `4:5`, and `16:9` crops no longer always cut from the center.
- Added horizontal and vertical crop sliders in the editor crop dock when a fixed ratio is active.
- Kept light/color/detail sliders paint-first; only crop geometry continues to trigger rendered preview work.
- Added a render regression test proving left/right crop positions bake different output.
- Updated the app version to `0.3.9+17`.

## Product truth

- This is not full drag-handle crop yet. It is the first working crop-position slice so users can control the crop result instead of getting a blind center crop.
- The next crop pass should add direct frame handles and straighten after real-device responsiveness is checked.
- AI upscale remains local fast/enhanced upscale only; no neural model claim was added.
