# OmniaPixels 0.3.7+15

Date: 2026-04-28

## Theme

Bug and performance hardening after live app smoke.

## What was bad

- The Android picker path could leave the app looking stuck on `Opening picker...` even though the user was inside or returning from the system picker.
- The busy overlay did not absorb taps, so users could keep touching navigation and controls while render/save/picker work was active.
- The Edit tab looked interactive even when no photo was loaded, which made disabled or irrelevant controls feel broken.
- The editor control dock was too cramped near the bottom navigation on tall phone layouts.
- Gallery load-more rebuilt the loaded asset list by copying all previous entries, which is the wrong direction for large libraries.

## What changed

- Split picker flow into native picker open and selected-photo preparation, with clearer status and error handling.
- Busy overlay now blocks underlying taps, and bottom navigation is disabled during active work.
- Edit tab now shows a dedicated empty state with `Pick Photo` and `Open Gallery`; editor tools appear only after a photo is loaded.
- Increased the editor control dock height so crop, brush, text, sticker, and light/color controls have room to breathe.
- Gallery pagination now appends assets in place, wraps thumbnail tiles in repaint boundaries, and keeps thumbnail decode sizing explicit.
- Bumped Flutter app version to `0.3.7+15`.

## Product truth

- This release fixes visible broken behavior and performance structure; it is not a final Play Store candidate.
- Upscale remains local and experimental in product quality. No real neural model is claimed in this build.
- Large-library performance still needs real-device proof on 10k and 25k+ photo libraries.
