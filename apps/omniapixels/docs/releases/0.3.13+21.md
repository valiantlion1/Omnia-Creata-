# OmniaPixels 0.3.13+21

Date: 2026-04-29

## Theme

Make crop behave like a real mobile photo editor tool instead of a coarse position-slider mock.

## References

- Adobe Lightroom Android crop/geometry direction: crop ratios, rotate, flip, and straighten-style geometry controls.
- Google Photos Android edit direction: crop/rotate as a normal photo-editing surface.
- Snapseed direction: dedicated crop and rotate/perspective tools in an image-first editor workflow.
- Flutter implementation package: `crop_your_image` `2.0.0`, used for embedded crop handles, crop rect movement, aspect ratio control, and crop backend behavior.

## What changed

- Added `crop_your_image` as the crop interaction engine.
- Replaced weak `Horizontal` / `Vertical` crop sliders with a direct handle-based crop frame on the image.
- Added crop modes: `Original`, `Free`, `1:1`, `4:5`, `9:16`, `4:3`, and `16:9`.
- Added crop-side geometry actions for full-frame reset, horizontal flip, left rotate, and right rotate.
- Added normalized crop frame state (`cropLeft`, `cropTop`, `cropWidth`, `cropHeight`) so crop can be exported from the same visible frame.
- Added a crop-base render path so crop editing can show the correct geometry base without continuously CPU-rendering while the crop frame moves.
- Updated export/render to bake normalized crop frames, flip, and rotate into the saved output.
- Added render coverage for direct crop-frame baking.
- Updated the app version to `0.3.13+21`.

## Product truth

- This is a crop-quality slice, not a full editor rewrite.
- Crop now has real handles and aspect behavior, but real-device touch feel still needs phone smoke on high- and mid-range Android devices.
